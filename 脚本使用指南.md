# Hexo 脚本使用指南 (Mac)

## 📁 脚本文件说明

我们已经创建了两个便捷脚本：

- **`dev.sh`** - 本地开发脚本 (clean → generate → server)
- **`deploy.sh`** - 部署脚本 (clean → generate → deploy)

## 🚀 运行方法

### 方法一：直接执行（推荐）

```bash
# 本地开发
./dev.sh

# 部署到 GitHub Pages
./deploy.sh
```

### 方法二：使用 bash 命令

```bash
# 本地开发
bash dev.sh

# 部署
bash deploy.sh
```

### 方法三：使用 sh 命令

```bash
# 本地开发
sh dev.sh

# 部署
sh deploy.sh
```

## 📍 操作步骤

### 1. 打开终端

**方法A：通过 Finder**
- 打开 Finder
- 导航到 `/Users/chenyinuo/VScode/Python/myblog/Blogfiles`
- 右键点击文件夹
- 选择 "服务" → "新建位于文件夹位置的终端窗口"

**方法B：通过终端**
```bash
cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles
```

**方法C：通过 VSCode**
- 在 VSCode 中打开项目
- 按 `Ctrl + `` (反引号) 打开终端
- 确保终端路径在 `Blogfiles` 目录

### 2. 运行脚本

**本地开发流程：**
```bash
./dev.sh
```

执行后会：
1. 清理缓存文件
2. 生成静态网站
3. 启动本地服务器 (http://localhost:4000)
4. 可访问管理后台 (http://localhost:4000/admin)

**部署流程：**
```bash
./deploy.sh
```

执行后会：
1. 清理缓存文件
2. 生成静态网站
3. 部署到 GitHub Pages
4. 网站将在 https://cccignore.github.io/ 更新

## 🛠️ 故障排除

### 权限问题

如果遇到 "Permission denied" 错误：

```bash
# 给脚本添加执行权限
chmod +x dev.sh deploy.sh

# 然后重新运行
./dev.sh
```

### 路径问题

确保在正确的目录下运行：

```bash
# 检查当前路径
pwd

# 应该显示：/Users/chenyinuo/VScode/Python/myblog/Blogfiles

# 如果不对，切换到正确路径
cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles
```

### 脚本不存在

检查脚本文件是否存在：

```bash
# 列出当前目录文件
ls -la *.sh

# 应该看到：
# -rwxr-xr-x  1 user  staff  xxx  date  dev.sh
# -rwxr-xr-x  1 user  staff  xxx  date  deploy.sh
```

## 📱 快捷使用技巧

### 1. 创建别名（可选）

在 `~/.zshrc` 文件中添加：

```bash
# Hexo 快捷命令
alias hexo-dev="cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles && ./dev.sh"
alias hexo-deploy="cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles && ./deploy.sh"
```

然后重新加载配置：
```bash
source ~/.zshrc
```

之后可以在任何目录直接运行：
```bash
hexo-dev      # 启动本地开发
hexo-deploy   # 部署博客
```

### 2. 使用 Finder 快速访问

1. 在 Finder 中将 `Blogfiles` 文件夹拖到侧边栏
2. 右键点击脚本文件
3. 选择 "打开方式" → "终端"

### 3. 创建桌面快捷方式

创建 `.command` 文件：

```bash
# 创建本地开发快捷方式
echo '#!/bin/bash
cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles
./dev.sh' > ~/Desktop/Hexo开发.command
chmod +x ~/Desktop/Hexo开发.command

# 创建部署快捷方式
echo '#!/bin/bash
cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles
./deploy.sh' > ~/Desktop/Hexo部署.command
chmod +x ~/Desktop/Hexo部署.command
```

## ⚠️ 注意事项

1. **本地开发脚本** (`dev.sh`) 会启动服务器，按 `Ctrl+C` 停止
2. **部署脚本** (`deploy.sh`) 需要网络连接和正确的 GitHub 配置
3. 脚本执行过程中如果出错会自动停止并显示错误信息
4. 部署后通常需要等待 1-5 分钟才能在网站上看到更新

## 🎯 常用工作流程

### 写作流程
1. 运行 `./dev.sh` 启动本地服务器
2. 访问 http://localhost:4000/admin 在线编辑
3. 或者直接编辑 `source/_posts/` 下的 markdown 文件
4. 在浏览器中预览效果
5. 满意后按 `Ctrl+C` 停止服务器
6. 运行 `./deploy.sh` 部署到线上

### 快速发布流程
1. 编辑文章
2. 直接运行 `./deploy.sh`
3. 等待部署完成
