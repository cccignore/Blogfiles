# hexo-admin 使用指南

## 🎉 安装成功！

hexo-admin 插件已经成功安装并运行！现在您可以直接在网页界面编写和管理博客了。

## 🚀 快速开始

### 访问管理后台

**管理后台地址**：http://localhost:4000/admin/

### 基本操作流程

1. **启动服务器**
   ```bash
   cd /Users/chenyinuo/VScode/Python/myblog/Blogfiles
   npx hexo server -d
   ```

2. **访问管理界面**
   - 在浏览器打开：http://localhost:4000/admin/
   - 您会看到一个现代化的管理界面

3. **创建新文章**
   - 点击 "New Post" 按钮
   - 填写文章标题
   - 设置分类和标签
   - 在左侧编辑器写作，右侧实时预览

4. **管理现有文章**
   - 在文章列表中查看所有文章
   - 点击文章标题进行编辑
   - 支持删除、发布、保存为草稿

## 🚀 发布流程

### 方式一：手动发布（推荐）
1. 在 admin 界面完成文章编写
2. 保存文章（自动保存到 `source/_posts/`）
3. 在终端执行：
   ```bash
   npx hexo generate
   npx hexo deploy
   ```

### 方式二：配置一键发布
1. 在 `_config.yml` 中添加：
   ```yaml
   admin:
     deployCommand: './hexo-deploy.sh'
   ```

2. 创建部署脚本 `hexo-deploy.sh`：
   ```bash
   #!/bin/bash
   npx hexo generate
   npx hexo deploy
   ```

3. 给脚本执行权限：
   ```bash
   chmod +x hexo-deploy.sh
   ```

4. 在 admin 界面点击 "Deploy" 按钮一键发布

## ⚠️ 注意事项

### 图片处理
- 图片会自动保存到 `source/images/` 目录
- 如果图片显示异常，检查路径格式
- 建议使用相对路径引用图片

### 中文支持
- 文章标题支持中文
- URL 可能仍为中文，建议手动设置 permalink
- 字数统计可能不准确（仅统计英文单词）

### 性能优化
- 大量文章时加载可能较慢
- 建议定期清理草稿和无用文件
- 图片文件不要过大

## 🛠️ 故障排除

### 常见问题

**Q: 无法访问 /admin/ 页面？**
A: 确保 hexo-admin 插件已正确安装，重启服务器

**Q: 图片无法显示？**
A: 检查图片路径，确保使用正确的相对路径格式

**Q: 保存后文章没有出现？**
A: 检查 `source/_posts/` 目录，确认文件已生成

**Q: Deploy 按钮不工作？**
A: 检查 `_config.yml` 中的 deployCommand 配置

### 重启服务
如果遇到问题，可以重启 hexo 服务：
```bash
# 停止服务（Ctrl+C）
# 重新启动
npx hexo server -d
```
